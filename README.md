## Prerequisites
1. Have STM32CubeIDE and STM32CubeMX set up and installed

## Workflow
1. Create a new branch and name it one of the following
    - `driver/driver-name` where "driver-name" will be replaced with whatever device your making a driver for
    - `feature/feature-name` where "feature-name" will be replaced with whatever feature you are testing
2. Create and test your feature in your new branch! Try to follow the existing file structure as best as possible (or create a new one for your purpose if necessary)
3. Once your features is near completion you can create a PR and ping someone for a review!
